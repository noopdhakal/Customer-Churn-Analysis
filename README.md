## 🚀 Project Overview

Customer churn is one of the most critical challenges in the telecom industry. This project focuses on building a **machine learning system** that predicts whether a customer is likely to leave (churn) or stay.

The goal is to help businesses take proactive actions to **retain customers and reduce revenue loss**.

## 🎯 Objectives

- Predict customer churn using machine learning models
- Identify key factors influencing churn
- Handle real-world challenges like **class imbalance**
- Build a deployable ML application


## 🛠️ Tech Stack

|Category|Tools Used|
|---|---|
|Language|Python|
|ML Libraries|Scikit-learn, Imbalanced-learn|
|Optimization|Optuna|
|Visualization|Matplotlib, Seaborn|
|Web Framework|Streamlit|
|Deployment|AWS EC2|
## 📈 Results

- Improved model performance using resampling techniques
- Achieved better recall for churn prediction
- Successfully deployed real-time prediction system


## 💻 How to Run

### 1. Clone the repository

```bash
git clone https://github.com/noopdhakal/Customer-Churn-Analysis.git
cd Customer-Churn-Analysis
```

### 2. Create environment

```bash
conda create -n churn python=3.9
conda activate churn
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Run notebook / app

```bash
jupyter notebook
```

or (if Streamlit app exists):

```bash
streamlit run app.py
```
